# [Flutter/Sqflite] DB 작업을 200% 효율적으로: Batch란 무엇이고 왜 써야 할까?

Flutter에서 로컬 데이터베이스로 `sqflite`를 사용하다 보면 `Batch`라는 단어를 마주치게 됩니다. 이름만 들어서는 "뭔가를 한꺼번에 처리하는 것" 같긴 한데, 정확히 무엇이고 왜 사용해야 할까요? 그냥 명령어를 하나씩 실행하는 것과 무엇이 다를까요?

결론부터 말하면, `Batch`는 데이터베이스 작업의 **성능**과 **안정성**을 극적으로 향상시키는 핵심 도구입니다. 이 글에서는 `Batch`의 개념과 장점을 쉬운 비유와 실제 코드를 통해 알아보겠습니다.

## 1. Batch란? 쇼핑 카트 🛒를 기억하세요

`Batch`는 **"여러 개의 데이터베이스 작업을 하나의 묶음(Transaction)으로 만들어 한 번에 처리하는 도구"**입니다.

가장 쉬운 비유는 마트의 쇼핑 카트입니다.

*   **Batch를 사용하지 않을 때:** 필요한 물건(우유, 계란, 빵)을 하나씩 따로 계산하는 것과 같습니다.
    1.  우유를 집어 계산하고, 다시 매장으로 돌아옵니다. (DB 작업 1번)
    2.  계란을 집어 계산하고, 다시 매장으로 돌아옵니다. (DB 작업 2번)
    3.  빵을 집어 계산하고, 쇼핑을 마칩니다. (DB 작업 3번)
    매우 비효율적이고 번거롭습니다.

*   **Batch를 사용할 때:**
    1.  **쇼핑 카트(`Batch`)를 준비합니다.**
    2.  필요한 물건(우유, 계란, 빵)을 **모두 카트에 담습니다.**
    3.  모든 물건을 다 담았으면, 계산대로 가서 **한 번에 결제합니다.**

당연히 후자가 훨씬 효율적입니다. `Batch`는 바로 이 쇼핑 카트와 같은 역할을 합니다.

## 2. Batch를 사용해야 하는 이유

`Batch`는 크게 두 가지 강력한 장점을 제공합니다.

### 가. 성능 향상 (Performance)

데이터베이스에 명령을 한 번 보낼 때마다 눈에 보이지 않는 통신 비용(오버헤드)이 발생합니다. 여러 개의 작업을 개별적으로 실행하면 이 비용을 매번 지불해야 합니다.

`Batch`는 여러 명령을 하나의 요청으로 묶어서 보내기 때문에, 이 통신 비용을 딱 한 번만 지불하게 됩니다. 특히 여러 테이블을 생성하거나 많은 양의 초기 데이터를 삽입하는 작업은 `Batch` 사용 여부에 따라 속도 차이가 크게 날 수 있습니다.

### 나. 원자성 (Atomicity)과 데이터 무결성

이것이 안정성 측면에서 가장 중요한 장점입니다. `Batch`로 묶인 작업들은 **"전부 성공하거나, 하나라도 실패하면 전부 실패(취소)하거나"** 둘 중 하나의 결과만을 보장합니다. 이를 **원자적(Atomic) 트랜잭션**이라고 합니다.

예를 들어, `users` 테이블을 만들고, 이어서 `categories` 테이블을 만드는 작업을 `Batch`로 묶었다고 가정해 봅시다. 만약 `categories` 테이블을 만드는 SQL 구문에 오타가 있어 실패한다면 어떻게 될까요?

*   **`Batch`가 없다면:** `users` 테이블만 덩그러니 만들어진, 비정상적인 상태의 데이터베이스가 남게 됩니다.
*   **`Batch`가 있다면:** `categories` 생성에 실패하는 순간, 이전에 성공했던 `users` 테이블 생성 작업까지 **모두 없었던 일로 되돌려 버립니다(Rollback)**. 데이터베이스는 작업 시작 전의 깨끗한 상태로 완벽하게 유지됩니다. 데이터의 일관성과 무결성을 지킬 수 있는 것입니다.

## 3. 실제 코드 예시

MoneyFit 프로젝트의 `database_helper.dart` 파일에서 데이터베이스를 처음 생성하는 `_onCreate` 메서드는 `Batch`의 좋은 활용 예시입니다.

```dart
// lib/core/database/database_helper.dart

Future<void> _onCreate(Database db, int version) async {
  // 1. 빈 쇼핑 카트(Batch)를 준비합니다.
  final batch = db.batch();

  // 2. 카트에 작업을 담습니다. (아직 실행은 안 함)
  //    - "테이블들 만들기"라는 작업을 담고,
  _createTables(batch);
  //    - "기본 카테고리 데이터 넣기"라는 작업을 담습니다.
  _seedDatabase(batch);

  // 3. 카트에 담은 모든 작업을 한 번에 실행(계산)합니다.
  //    이때 하나라도 실패하면 모든 작업이 자동으로 취소됩니다.
  await batch.commit();
}
```

위 코드에서 `batch.commit()`이 호출되기 전까지는 실제 데이터베이스에 아무런 변화도 일어나지 않습니다. `commit()`이 호출되는 순간, 묶여있던 모든 작업이 하나의 트랜잭션으로 실행됩니다.

## 결론

`Batch`는 단순히 코드를 보기 좋게 묶는 기능이 아닙니다. 여러 개의 데이터베이스 쓰기(Create, Update, Delete) 작업을 수행할 때는 `Batch`를 사용하는 것이 좋습니다.

*   **성능:** 불필요한 오버헤드를 줄여 앱을 더 빠르게 만듭니다.
*   **안정성:** 예측 불가능한 오류가 발생했을 때 데이터가 깨지는 것을 막아줍니다.

`sqflite`를 사용한다면, `Batch`를 적극적으로 활용하여 더 효율적이고 안정적인 애플리케이션을 만들어 보세요.
