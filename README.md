# MoneyFit

MoneyFit은 사용자가 자신의 소비 습관을 쉽게 파악하고 관리할 수 있도록 돕는 가계부 앱입니다. 일일 예산 혹은 월간 예산을 설정하여 당일 지출이 가능한 자율 지출을 파악할 수 있고, 캘린더,통계,지출 내역를 통해 소비 패턴을 한눈에 파악할 수 있습니다.

## 주요 기능

### 0. 온보딩

- 초기 데이터 설정 : 앱 기능 사용에 필요한 일일 혹은 월간 예산을 설정합니다.

### 1. 메인 화면

- 일일/월간 지출 현황: 오늘 사용한 금액과 일일 기준,월간 기준 사용자 선택에 따라 남은 예산을 원형 프로그레스바로 시각화하여 보여줍니다.
- 월평균 변동 지출 및 연속 목표 성취일: 월별 평균 지출액과 연속으로 목표를 달성한 날짜를 표시하여 소비 습관을 관리하는 데 동기를 부여합니다.
- 오늘의 지출 보기: 오늘의 지출 내역을 간략하게 확인할 수 있는 바텀 시트 모달을 제공합니다.
- 지출 등록: 새로운 지출을 간편하게 등록할 수 있는 바텀 시트 모달 형식의 입력 폼을 제공합니다.

### 2. 캘린더

- 월별 지출 내역: 캘린더 형식으로 일별 지출액을 표시하여 한 달의 소비 흐름을 쉽게 파악할 수 있습니다.
- 월간 통계: 월간 변동 지출, 필수 지출, 목표 성공/실패/연속 성공일 통계를 제공하여 성과를 추적할 수 있습니다.
- 일별 상세 내역: 특정 날짜를 선택하면 해당일의 바텀 시트 모달 형식의 상세 지출 내역을 확인할 수 있습니다.

### 3. 통계

- 월간 카테고리별 지출 파이 차트 형식으로 제공
- 월간 지출 top3카테고리 제공

### 4. 지출 내역

- 월간 모든 지출 내역 수직 스크롤뷰로 확인 가능
- 지출 내역 내 필터링 가능

# 앱 운영 데이터 확인

250925 ~ 251022 기준으로 리텐션 여부를 확인해봤다.
사용자로 판단하는 기준은 이렇게 3개로 추적해봤다.

1. 앱 설치 후 첫 번째 터치
   9.25 ~ 9.27 6명
   9.28 ~ 10.4 14명
   10.5 ~ 10.11 17명
   10.12 ~ 10.18 10명
   10.19 ~ 10.22 3명
2. 첫 번째 터치 후 듀토리얼 완료
   9.25 ~ 9.27 2명
   9.28 ~ 10.4 2명
   10.5 ~ 10.11 7명
   10.12 ~ 10.18 3명
   10.19 ~ 10.22 0명
3. 듀토리얼 완료 후 지출 기록
   9.25 ~ 9.27 1명
   9.28 ~ 10.4 0명
   10.5 ~ 10.11 0명
   10.12 ~ 10.18 1명
   10.19 ~ 10.22 0명

우선 이렇게 데이터를 확인해봤을 때 앱을 설치하고 실행한 인원은 50명
그 중 듀토리얼은 완료한 사람은 14명 , 지출을 실제로 등록한 사람은 2명이다.
결국에 사용자가 앱을 실행했을 때, 지출을 등록할 이유를 못느꼈다는 것이다.
그렇다면 사람들에게 이 앱을 사용해야 하는 이유에 대해서 알려줘야 한다.
우선 가장 중요한 것은 사용자들이 지출을 등록하게 하고, 자신이 사용할 수 있는 금액이 줄어든다는 것을 확인할 수 있어야 한다. 결국 이를 통해 내가 이번 달 얼마를 더 사용할 수 있구나라는 정보를 중시하는 사람들이 남고 그렇지 않은 사람이 떠나는 것이다. 결국 있어야 하는 것만 남기는 것도 매우 중요한 요소인 것 같다.
그리고 자연스럽게 행동을 유도해야 하는 것도 있는 것 같다.
다운로드 수가 많은 2개의 앱을 확인해봤는데,

1. 오픈앱 광고가 없었다.
2. 온보딩 단계에서 따로 앱에 대한 설명이 없었고, 즉시 앱을 확인할 수 있었다.
3. 사용자가 디자인을 커스텀 할 수 있었다.

이 외에는 내 앱에서 개선해야한다고 느낀 부분이 없었다.

나는 그러면 어떻게 해야할까?
결국 앱 초기 설치부터 앱 핵심 가치를 전달하는 과정을 줄여야겠다고 생각한다.

1. 애초에 온보딩이 길다고 느끼지는 않지만, 그럼에도 최대한 바로 앱을 사용할 수 있도록 온보딩 단계를 줄여본다.
2. 앱 소개 온보딩을 없애고, 즉시 유저에게 예산을 물어본다. 근데 어쩌면, 이 한 달에 얼마를 쓸 수 있는지 물어보는 것이 유저에게 고민을 남기고 흐름을 끊는 것은 아닐까?
3. 앱을 켰을 때 실행되는 광고를 제거한다.
4. 사용자가 디자인 색상을 커스텀 할 수 있도록 변경한다.

## 앱 개선사항

현재 앱 실행 흐름

1. 온보딩 과정을 줄인다. 기존 온보딩1~3 => 예산 설정 과정을 즉시 예산 설정 화면으로 라우팅하도록 변경
2. 사용자가 디자인 색상을 커스텀 할 수 있도록 변경한다.
   - 기존 메인 색상 하나에 테마가 모두 맞춰져있었는데, 이 부분을 유저가 색상을 선택할 수 있고, 또한 다크모드도 설정할 수 있돌고 하면 좋을 것이다.
   - 이를 테스트하기 위해서 앱에 적용해보기 이전에 색상을 바꾸는 프로젝트를 우선 진행한다.
