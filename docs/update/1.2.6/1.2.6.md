# 1.4.7 업데이트

## 개요

방치해뒀던 앱이 1월15일 기준 갑자기 지난 7일간 많은 사람들이 유입되었다.

## Part 1: 테마 시스템 리팩토링

### 문제 인식

기존 테마 시스템에는 다음과 같은 구조적 문제가 있었다:

1. **하드코딩된 색상 분산**: `LightAppColors`, `DarkAppColors` 클래스에 색상이 static const로 정의되어 있어, 런타임에 색상 변경이 불가능했다.
2. **테마 확장성 부재**: 사용자가 원하는 색상으로 앱 테마를 커스터마이징할 수 없었다.
3. **색상 접근 방식의 비일관성**: 일부는 `LightAppColors.primary`로, 일부는 `Theme.of(context).colorScheme.primary`로 접근하는 등 혼재되어 있었다.
4. **TextStyle에 색상 하드코딩**: `AppTextStyles`에 색상이 포함되어 있어 테마 변경 시 텍스트 색상이 따라가지 않았다.

### 해결 방안

Flutter의 `ThemeExtension` 메커니즘을 활용한 완전한 커스텀 테마 시스템을 구축했다.

#### 핵심 아키텍처

```
User Action (색상 선택)
    ↓
ThemeNotifier (Riverpod)
    ↓
AppThemeGenerator.fromSeed(colorSeed)
    ↓
AppThemeColors (ThemeExtension)
    ↓
ThemeData
    ↓
MaterialApp
    ↓
UI 리빌드
```

#### 주요 구현 내용

| 파일 | 역할 |
|------|------|
| `app_theme_colors.dart` | 22개 의미론적 색상 속성을 가진 ThemeExtension |
| `app_theme_generator.dart` | colorSeed → Light/Dark 테마 자동 생성 |
| `theme_extensions.dart` | `context.colors`, `context.textTheme` 등 간결한 접근 |
| `theme_provider.dart` | Riverpod 기반 테마 상태 관리 |
| `theme_repository.dart` | SharedPreferences 영구 저장 |

#### 색상 매핑 (Before → After)

| 기존 | 신규 | 용도 |
|------|------|------|
| `LightAppColors.primary` | `context.colors.brandPrimary` | 버튼, 선택된 네비게이션 |
| `LightAppColors.background` | `context.colors.screenBackground` | Scaffold 배경 |
| `LightAppColors.textPrimary` | `context.colors.textPrimary` | 주요 텍스트 |
| `LightAppColors.textOnPrimary` | `context.colors.textOnBrand` | 브랜드 색상 위 텍스트 |

### 사용자 커스터마이징 기능

1. **프리셋 색상**: 8개 기본 색상 제공 (브라운, 블루, 그린, 레드, 퍼플, 오렌지, 시안, 핑크)
2. **HSV 컬러 피커**: 완전 자유로운 색상 선택
3. **최근 사용 색상**: 최대 8개 저장
4. **실시간 미리보기**: 선택 전 테마 프리뷰 제공

### 폰트 크기 커스터마이징

| 옵션 | 스케일 | 용도 |
|------|--------|------|
| Small | 0.85x | 더 많은 정보 표시 |
| Medium | 1.0x | 기본값 |
| Large | 1.15x | 가독성 향상 |

### 테스트 커버리지

- **Property-Based Tests**: 색상 일관성, 대비율, 라운드트립 검증
- **Unit Tests**: 각 컴포넌트별 동작 검증
- **총 50+ 테스트 케이스** 작성

### 마이그레이션 전략

1. 기존 `LightAppColors`/`DarkAppColors`를 `@Deprecated` 처리
2. 점진적으로 `context.colors.xxx` 방식으로 전환
3. 모든 주요 화면(Home, Calendar, Expense, Settings, Onboarding) 마이그레이션 완료

## Part 2: 유럽 시장 확장 (언어 현지화)

방치해뒀던 앱이 1월15일 기준 갑자기 지난 14일간 많은 사람들이 유입되었다.
특히 스페인, 폴란드, 우크라이나 등 다양한 유럽 국가에서 유입이 확인되었다.
현재 앱은 영어, 한국어, 필리핀어, 인도네시아어만 지원하고 있어, 신규 유입 사용자들의 언어 지원이 시급하다.
단순 유입 수뿐만 아니라, **참여율(Engagement Rate)**과 **평균 참여 시간**을 종합적으로 고려하여 업데이트 우선순위를 선정한다.

## 우선순위 도출 및 분석

사용자 수(양적 지표)와 참여율(질적 지표)을 종합하여 다음과 같이 우선순위를 도출했다.

1.  **최우선 지원 (Must Have)**: **스페인, 폴란드**
    - 압도적인 사용자 수와 높은 참여율을 기록. 앱 성장에 가장 중요한 그룹.
2.  **우선 지원 (Should Have)**: **우크라이나, 체코**
    - 준수한 사용자 규모와 높은 참여율. 특히 우크라이나는 참여율이 매우 높아 언어 지원 시 체류 시간 증대가 기대됨.
3.  **잠재적 중요군 (High Potential)**: **미국, 이탈리아**
    - 사용자 수는 적지만 참여율과 체류 시간이 매우 높음(미국 3분 16초). 충성도 높은 사용자로 전환될 가능성 높음. (미국은 이미 영어 지원 중)
4.  **관찰 및 후순위**: **독일, 로마니아, 슬로바키아, 불가리아**
    - 독일과 로마니아는 사용자 수 대비 참여율이 50%대로 낮아, 단순 번역만으로는 이탈을 막기 어려울 수 있음. 우선순위에서 다소 밀림.

## 업데이트 계획

1.  **신규 언어 추가 (우선순위 순)**
    - **1차**: 스페인어(es), 폴란드어(pl)
    - **2차**: 우크라이나어(uk), 체코어(cs)
    - **3차**: 독일어(de), 이탈리아어(it) (독일은 낮은 참여율 개선 실험, 이탈리아는 고참여율 지원)
2.  **전략**
    - gemini를 통해 `l10n` arb 파일을 생성하여 빠르게 텍스트 번역 적용하고 품질 검수에 신경쓸 것
3.  **설정 화면에 언어 변경 아이콘 제공**
    - 기존 사용자들에게 갑작스러운 언어 변경이 당황스러울 수 있으므로, 업데이트 안내문과 설정화면에 언어 변경 시스템을 제공한다.
